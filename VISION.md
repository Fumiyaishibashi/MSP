# MBS Synergy Platform - 拡張ビジョン

## 📊 目標

**組織全体の「人」と「外部リソース」を可視化し、初めましての繋がりを増やす**

過去のプロジェクトで活躍した「人」や「外部会社」の情報が失われず、次のプロジェクトで再発見・再利用される組織へ。

---

## 🏗️ アーキテクチャ

### 4種類のノート（1つのキャンバス内に混在）

| ノート種別 | 色 | 用途 | 配置例 |
|----------|-----|------|--------|
| **IP ノート** | 黄色 | 番組、キャラクター、技術リソース | キャンバスに配置 |
| **アイデア ノート** | ピンク | 企画案、自由記述メモ | キャンバスに配置 |
| **人メモ** | 青色 | 社内メンバー（専門領域、過去実績） | キャンバス + サイドバー |
| **会社メモ** | 緑色 | 外部企業（実績、推薦者） | キャンバス + サイドバー |

## 💬 チャット機能

### 仕様

**チャットスペース:**
- メモごとに専用チャットスレッド
- **オープンなスペース** = 誰でもコメント可能
- コメント履歴は永続保持（次回アクセス時も見える）

**チャット表示例:**

```
【Aさん - グッズ制作の達人】- チャット

2025-11-28 10:00 | 鈴木
  Aさん、ごぶごぶ FES のグッズ制作を手伝ってもらえますか？

2025-11-28 10:15 | Aさん（Aさん）
  もちろんです！どんなグッズですか？

2025-11-28 10:20 | 佐藤
  俺も参加します。去年のヤンタングッズのノウハウ活かしましょう

2025-11-28 11:00 | Aさん（Aさん）
  佐藤さんもいいね。では打ち合わせ日程を決めましょう

[💬 コメント入力欄]
[送信ボタン]
```

**特徴:**
- 「誰が書いたか」が見える（著者表示）
- スレッド形式で時系列に整理
- メモのコンテキストと一緒に管理

## 📊 データ型定義（イメージ）

### 人メモ（PersonMemo）

```typescript
interface PersonMemo {
  id: string;
  name: string;              // 「Aさん」
  department: string;        // 「グッズ制作部」
  expertise: string[];       // ["グッズ制作", "ブランディング"]
  email: string;             // a@mbs.co.jp
  phone?: string;            // 090-XXXX-XXXX
  pastProjects: string[];    // ["ごぶごぶ FES", "ヤンタン周年"]
  avatar?: string;           // アバター画像URL
}
```

### 会社メモ（CompanyMemo）

```typescript
interface CompanyMemo {
  id: string;
  name: string;              // 「〇〇デザイン」
  specialty: string[];       // ["グラフィック", "Web制作"]
  pastProjects: string[];    // ["ごぶごぶ FES", "ラジオステーション"]
  pointOfContact: {           // 推薦者・担当者（複数可）
    personId: string;
    name: string;
    email: string;
    role?: string;            // 「グッズ制作部」など
  }[];
}
```

### チャット（Message）

```typescript
interface Message {
  id: string;
  memoType: 'person' | 'company';  // どのメモに紐付いているか
  memoId: string;                   // メモのID
  author: string;                   // 「鈴木」
  content: string;                  // メッセージ本文
  timestamp: Date;                  // 2025-11-28 10:00
  reactions?: string[];             // 👍 とか
}
```

## 🔄 ワークフロー例

### シナリオ：ごぶごぶ FES のグッズ制作

**1️⃣ 企画初期段階**
- キャンバスに「ごぶごぶ FES」企画がある
- 「グッズ制作が必要」というアイデアメモを追加

**2️⃣ 人・会社メモを配置**
- サイドバーから「Aさん（グッズ制作の達人）」をドラッグ
- キャンバスに配置（青色ノート）
- サイドバーから「〇〇デザイン（グラフィック得意）」をドラッグ
- キャンバスに配置（緑色ノート）

**3️⃣ 詳細確認**
- Aさんノートをクリック
- 「メールアドレス: a@mbs.co.jp」確認
- 「過去実績: ヤンタン周年グッズ」確認
- → 「あ、この人なら信頼できる」

**4️⃣ チャット開始**
- 「💬 チャットを開く」をクリック
- Aさんに提案「グッズ制作お願いできますか？」
- Aさんが返信「いいですよ。こんなの作れます」
- 他の人（佐藤さん）も参加「俺も一緒にやります」
- オープンなスペースなので、後から見た人も「あ、これやってるんだ」と発見

**5️⃣ 後発人材の参画**
- 新入社員が企画を見て
- 「あ、Aさんが関わってるのか。去年のグッズやった人だ」
- チャットで「初心者ですが手伝えることあります？」
- → 初めまして、新しい繋がり生成

## ✨ このプラットフォームの価値

| 現状 | 拡張後 |
|------|--------|
| イベント企画の「内容」だけが見える | イベント企画 + 「人」「会社」が見える |
| グッズ制作の成功理由が記録されない | 「Aさんがいたからできた」が可視化される |
| 部門間で優秀な人の存在が知られない | 全社で専門家が発見できる |
| 後発プロジェクトが初めましてから始まる | 過去の成功事例を参考に再利用できる |
| チャットが分散（メール、LINE、Slack） | プロジェクトのコンテキスト内でチャット完結 |

## 💭 実装上の注意点

1. **チャットの「オープン性」**
   - 誰でもコメント可能 = 情報の透明性向上
   - ただし、プライベート情報は別管理

2. **推薦者複数**
   - 会社メモには複数の「担当者」「推薦者」がいる
   - 各推薦者に直接連絡できる仕様

3. **ドラッグ&ドロップ**
   - IP資産と同じく、人メモ・会社メモもドラッグ可能
   - 配置位置 = 関連性を示す（業界慣例的に）

4. **データ管理**
   - 人メモ・会社メモはマスターデータ
   - プロジェクト内に「配置」される（参照）
   - 本体の人メモ・会社メモを編集すれば、全プロジェクトに反映
